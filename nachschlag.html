🔹 HTML – Grundstruktur einer Seite

Jede HTML-Datei beginnt mit dem Doctype und hat einen Kopf (<head>) und einen sichtbaren Bereich (<body>).

<!DOCTYPE html>
<html lang="de">

  <head>
    <meta charset="UTF-8">       <!-- Zeichensatz UTF-8 -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Meine erste Seite</title>
  </head>

  <body>
    <h1>Willkommen</h1>
    <p>Dies ist meine erste Webseite.</p>
  </body>

</html>

✅ Elemente:
<h1> bis <h6> – Überschriften (H1 = wichtig)
<p> – Absatz
<a href=""> – Link
<br> – Zeilenumbruch
<hr> – Trennlinie

🔹 HTML – Listen, Links & Bilder

Listen ordnen Inhalte strukturiert. Links verbinden Seiten. Bilder zeigen Medien an.

📋 Ungeordnete Liste (ul)

<ul>
  <li>Apfel</li>
  <li>Banane</li>
  <li>Kirsche</li>
</ul>

📑 Geordnete Liste (ol)

<ol>
  <li>Erster Schritt</li>
  <li>Zweiter Schritt</li>
  <li>Dritter Schritt</li>
</ol>

🔗 Link zu einer anderen Seite

<a href="https://example.com" target="_blank">
  Zur Webseite
</a>

target="_blank" öffnet den Link in einem neuen Tab.

🖼️ Bild anzeigen

<img src="bild.jpg" alt="Beschreibung" width="300">
  • src = Bildpfad
  • alt = Beschreibung, falls Bild fehlt (auch für Barrierefreiheit)
  • width = Breite in Pixel (optional)

🔹 HTML – Formulare: Eingabefelder & Buttons

Formulare werden verwendet, um Daten vom Nutzer zu erfassen und an den Server zu senden. 

<form action="/ziel" method="post">

  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>

  <label for="email">E-Mail:</label>
  <input type="email" id="email" name="email" required>

  <label>
    <input type="checkbox" name="agb"> AGB akzeptieren
  </label>

  <input type="submit" value="Absenden">

</form>

✅ Erklärung:
action = URL, an die die Daten gesendet werden
method="post" = sendet die Daten im Hintergrund
label for + id = sorgt für bessere Bedienbarkeit

🔳 Feldgruppen mit <fieldset> und <legend>

<fieldset>
  <legend>Persönliche Daten</legend>

  <label for="stadt">Stadt:</label>
  <input type="text" id="stadt" name="stadt">

  <label for="plz">PLZ:</label>
  <input type="text" id="plz" name="plz">

</fieldset>

✅ Tipp:
Nutze <fieldset> + <legend> für sinnvolle Gruppierung – z. B. Kontaktdaten, Lieferadresse, etc.

🔹 HTML – Aufbau moderner Seiten mit Struktur-Tags

Semantische Tags beschreiben die Funktion des Inhalts – so verstehen Browser, Suchmaschinen und Screenreader besser, was gemeint ist.

  • <header> – oberer Seitenbereich (Logo, Navigation)
  • <nav> – Navigation/Links
  • <main> – Hauptinhalt der Seite
  • <section> – logischer Abschnitt
  • <article> – eigenständiger Inhalt (z. B. Blogbeitrag)
  • <footer> – unterer Seitenbereich (z. B. Impressum)
<header>
  <h1>Meine Webseite</h1>
  <nav>
    <a href="#start">Start</a>
    <a href="#kontakt">Kontakt</a>
  </nav>
</header>

<main>

  <section>
    <h2>Willkommen</h2>
    <p>Hier steht der Hauptinhalt.</p>
  </section>

  <article>
    <h3>Ein Beitrag</h3>
    <p>Dieser Artikel ist eigenständig.</p>
  </article>

</main>

<footer>
  <p>© 2025 Micha</p>
</footer>

✅ Praxis-Tipp:
Nutze <main> nur 1× pro Seite. Alles, was nicht zur Navigation gehört, kommt rein.

📘 Begriffserklärungen – Klassen & IDs im Code

Diese Liste erklärt dir die häufigsten Klassennamen, die du beim Coden siehst – kurz, präzise, realistisch.

  • .container
    → Umfasst den gesamten Inhalt (z. B. für max-width, zentriert)
  • .box
    → Eine einfache Box – oft für Inhalte mit Rahmen, Abstand, Farbe
  • .form-wrapper
    → Umschließt ein Formular zentriert mit Padding und max-width
  • .button-primary
    → Haupt-Button, optisch hervorgehoben (z. B. gelb, fett)
  • .hidden
    → Unsichtbar (z. B. mit display: none), oft bei Tabs verwendet
  • .inline-element
    → Kleines Element in einer Zeile, z. B. für Buttons oder Icons
  • .main-header
    → Obere Kopfzeile einer Seite (meist mit Logo + Navigation)
  • .item
    → Einzelnes Element in einer Liste oder Flexbox-Gruppe

✅ Praxis-Tipp:
Halte Klassennamen logisch und sprechend – du musst sie in 6 Monaten noch verstehen.

📘 Weitere Begriffe – realistisch & direkt aus dem Projekt

  • id="meinButton"
    → Wird im JS mit getElementById() angesprochen, z. B. für Klicks
  • .tab
    → Button, der per JavaScript einen Tab-Inhalt anzeigt (z. B. HTML, CSS, JS)
  • .tab-content
    → Container, der Inhalte für den gewählten Tab enthält – per Klasse „hidden“ ausblendbar
  • .code
    → Darstellung von Codeblöcken, meist mit dunklem Hintergrund & Monospace-Schrift
  • data-target="js"
    → Spezielles Attribut für JS-Auswahl (→ z. B. dataset.target)
  • .hidden
    → Wird vom Script gesetzt/entfernt, um Inhalte sichtbar oder unsichtbar zu machen
  • .gradient
    → Klasse mit background: linear-gradient(...); erzeugt weiche Farbverläufe
  • .highlight
    → Markiert wichtigen Text (z. B. background: yellow)

✅ Tipp:
IDs werden nur einmal verwendet – Klassen kannst du mehrfach einsetzen.
